然后确认微信开发者工具当前打开的项目目录，与 Codex 修改的仓库目录完全一致。

检查当前分支是否已经包含这些文案：

如代码仍出现“考虑下...再抽一次！”、记录页只有“去选饭”等旧内容，说明上一轮修改没有完整应用。请先补齐上一轮功能修改，再做本次布局修复。

## 2. 当前布局问题的根因

当前通用样式类似：

```css
.actions {
  display: flex;
  flex-wrap: wrap;
  gap: 16rpx;
}

.actions .btn {
  flex: 1 1 auto;
  min-width: 180rpx;
}
```

这不能保证横向排列，因为：

- `flex-wrap: wrap` 明确允许换行；
- `flex: 1 1 auto` 会根据文案计算初始宽度；
- `min-width: 180rpx` 加上间距和卡片内边距后容易超宽；
- 微信原生 `button` 自带 margin 和默认尺寸；
- 多个独立 `.actions` 容器本身就会形成上下排列。

本次不得只缩小字号或 gap，也不得继续使用通用 `.actions` 处理关键横向布局。

## 3. 必须改用专用布局类的区域

以下区域必须移除 `actions` 类，改为专用布局：

1. 选饭页：去管理选项 / 查看抽取记录
2. 做饭页：去管理菜谱 / 查看抽取记录
3. 记录页：去选饭 / 去做饭
4. 每条吃饭记录：编辑 / 删除
5. 本次结果：四个结果操作按钮
6. 最近抽取记录：记入记录 / 删除
7. 最近抽取记录标题：查看全部

## 4. 统一双列横向布局

新增类似样式：

```css
.two-column-actions {
  width: 100%;
  display: grid;
  grid-template-columns: repeat(2, minmax(0, 1fr));
  gap: 12rpx;
  box-sizing: border-box;
}

.two-column-actions .btn {
  width: 100%;
  min-width: 0;
  margin: 0;
  box-sizing: border-box;
  padding: 14rpx 10rpx;
  font-size: 24rpx;
  line-height: 1.35;
  white-space: normal;
}

.btn::after {
  border: none;
}
```

必须满足：

- 始终两列；
- 不使用 `flex-wrap`；
- 两列等宽；
- `min-width: 0`；
- 按钮 `margin: 0`；
- 按钮 `width: 100%`；
- 文案可在按钮内部换两行；
- 按钮本身不能换到下一整行；
- 常见窄屏仍保持双列。

## 6. 结果区域四按钮

四个按钮必须放在同一个网格容器中，不能拆成两个 `.actions`：

```xml

  <button class="btn">好，就吃这个，记下来！</button>
  <button class="btn">不算，再抽一次！</button>
  <button class="btn">暂记一下，再抽个别的吧~</button>
  <button class="btn">算了，不抽了~</button>
</view>
```

```css
.result-action-grid {
  display: grid;
  grid-template-columns: repeat(2, minmax(0, 1fr));
  gap: 12rpx;
  width: 100%;
}

.result-action-grid .btn {
  width: 100%;
  min-width: 0;
  min-height: 76rpx;
  margin: 0;
  padding: 12rpx 8rpx;
  font-size: 23rpx;
  line-height: 1.3;
  border-radius: 18rpx;
  white-space: normal;
  box-sizing: border-box;
}
```

验收：

- 第一行两个按钮；
- 第二行两个按钮；
- 不能出现一行一个；
- 文案完整；
- 选饭与做饭使用同一套样式。
