# Codex 开发任务：微信小程序《我要吃饭》

## 1. 任务目标

请在当前 Git 仓库中开发一个可直接导入微信开发者工具运行的原生微信小程序，应用名称暂定为 **“我要吃饭”**。

该小程序用于帮助用户决定“吃什么”，第一版包含两个核心功能：

1. **选饭**：从用户维护的若干自由文本选项中等概率随机抽取一个。
2. **做饭**：从用户维护的若干菜谱中等概率随机抽取一个，并展示菜谱详情。

同时实现：

- 自动保存抽取记录；
- 用户手动记录实际吃了什么；
- 本地数据管理；
- 基础设置；
- 默认示例数据；
- 类似老虎机的滚动抽取动画。

本项目是个人开发、轻量级 MVP。第一版不得引入后端、云数据库、登录、图片上传或复杂推荐算法。

## 5. 随机规则

第一版全部采用等概率随机。

### 5.1 选饭

候选集合：

```js
choices.filter(item => item.enabled)
```

### 5.2 做饭

候选集合：

```js
recipes.filter(item => item.enabled)
```

### 5.3 边界情况

必须正确处理：

- 没有可用候选项：
  - 不执行随机；
  - 显示明确提示；
  - 提供前往管理页的按钮。
- 只有一个候选项：
  - 可以播放简短动画；
  - 最终结果为该唯一候选项。
- 动画期间：
  - 禁止重复点击；
  - 禁止同时触发多个定时器；
  - 页面卸载时清理定时器。
- 每次正式抽取完成后：
  - 自动写入抽取记录。
- “再抽一次”：
  - 视为新的正式抽取；
  - 新增一条抽取记录。

使用 `Math.random()` 即可，不需要密码学随机数。

应将随机逻辑抽离为独立工具函数，以便后续扩展权重随机。

## 6. 类老虎机滚动动画

实现一个简洁的文字滚动/快速切换动画，视觉上类似老虎机，但不要实现复杂转盘。

要求：

- 动画总时长约 1.2–1.8 秒；
- 开始阶段切换较快；
- 结束阶段逐渐减速；
- 最终停在预先计算好的结果；
- 先计算最终结果，再播放动画；
- 动画过程不得改变最终概率；
- 动画期间主按钮禁用；
- 页面卸载时清理所有定时器；
- 避免重复定时器造成闪烁或内存泄漏；
- 不依赖第三方动画库；
- 可以使用轻微缩放或透明度变化，但保持简洁；
- 真机震动不是必需功能。

## 8. 本地存储

建议使用明确的 storage keys，例如：

```js
const STORAGE_KEYS = {
  META: "我要吃饭_meta",
  CHOICES: "我要吃饭_choices",
  RECIPES: "我要吃饭_recipes",
  DRAW_HISTORY: "我要吃饭_draw_history",
  MEAL_RECORDS: "我要吃饭_meal_records"
};
```

如果中文 key 可能造成维护不便，也可以统一使用英文前缀，例如：

```js
const STORAGE_KEYS = {
  META: "woya吃饭_meta"
};
```

更推荐纯英文且稳定的 key：

```js
const STORAGE_KEYS = {
  META: "meal_decider_meta",
  CHOICES: "meal_decider_choices",
  RECIPES: "meal_decider_recipes",
  DRAW_HISTORY: "meal_decider_draw_history",
  MEAL_RECORDS: "meal_decider_meal_records"
};
```

要求：

- 所有读取提供默认值；
- 捕获并处理读取异常；
- 写入失败时给用户明确提示；
- 防止损坏数据导致整个小程序无法进入；
- 初始化默认数据只执行一次；
- 用户删除默认数据后，下次启动不得自动重新生成；
- “恢复默认数据”由用户主动触发；
- “清空全部数据”必须二次确认；
- 清空后应回到可用状态，而不是页面报错；
- 所有数组返回前进行基本结构校验；
- 不要过度设计复杂 ORM 或状态管理器。

## 9. 北京时间规则

第一版固定使用：

- 时区：`Asia/Shanghai`
- UTC+8
- 北京时间

不得依赖设备当前时区决定记录所属日期和展示时间。

注意：微信小程序运行环境不一定完整支持所有 IANA 时区 API。请优先实现一个不依赖设备时区、可测试的 UTC+8 时间工具。

所有时间底层保存为毫秒时间戳。

### 9.1 北京时间转换

建立独立工具函数，至少支持：

- 获取当前北京时间的年、月、日、小时、分钟；
- 将用户选择的北京时间日期和时间转换为 UTC 时间戳；
- 将时间戳格式化为北京时间日期；
- 按北京时间日期生成分组 key；
- 按时间戳排序。

实现时需正确处理日期进位、月底、年底和闰年。

### 9.2 人格化时间

按照北京时间分钟数显示。

规则：

| 分钟范围 | 显示 |
|---|---|
| 00–19 | `X点多` |
| 20–39 | `X点半左右` |
| 40–59 | `不到下一点` |

采用 12 小时制。

示例：

```text
00:10 -> 12点多
00:30 -> 12点半左右
00:50 -> 不到1点

09:10 -> 9点多
09:30 -> 9点半左右
09:50 -> 不到10点

12:10 -> 12点多
12:30 -> 12点半左右
12:50 -> 不到1点

13:05 -> 1点多
13:30 -> 1点半左右
13:50 -> 不到2点

23:50 -> 不到12点
```

人格化时间只用于显示，不得作为原始时间数据保存。

第一版不添加：

- 上午；
- 下午；
- 晚上；
- 半夜；

但时间工具应保持易于扩展。
