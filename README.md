# 我要吃饭

原生微信小程序 MVP，用本地存储完成“选饭”和“做饭”随机抽取，并记录实际吃过什么。

## 功能

- 底部三个入口：选饭、做饭、记录
- 选饭页：从启用的候选项中等概率随机抽取，并单独保留最近记录与最后结果
- 做饭页：从启用的菜谱中等概率随机抽取，并保存完整快照与单独结果状态
- 抽取历史：按北京时间分组查看，并支持全部 / 选饭 / 做饭筛选
- 实际吃饭记录：按北京时间分组新增、编辑、删除
- 抽取结果可预填到记录编辑页，保存后写入 `sourceDrawId` 并防止重复记入
- 抽取页支持两种“再抽一次”语气，并区分保留当前记录或删除当前记录
- 选饭选项管理：新增、编辑、删除、启用/停用
- 菜谱管理：新增、编辑、删除、启用/停用
- 设置页：恢复默认数据、清空全部数据、关于页

在仓库根目录执行：

```bash
node tools/self-check.js
```

## 导入微信开发者工具

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录指向当前仓库根目录。
4. AppID 可使用当前 `project.config.json` 中的占位值，或替换为你自己的测试 AppID。
5. 导入后确认底部有“选饭”“做饭”“记录”三个 tab。

## 默认数据

首次启动会自动写入示例选项和示例菜谱。若你在设置页点击“恢复默认数据”，会重新覆盖为示例数据。

## 数据存储

数据保存在微信小程序本地存储中，键名如下：

- `meal_decider_meta`
- `meal_decider_choices`
- `meal_decider_recipes`
- `meal_decider_draw_history`
- `meal_decider_meal_records`

## 当前限制

- 没有登录、云同步或多人共享
- 没有图片上传
- 没有外部接口或 AI 推荐
- 只支持固定餐次

## 验证步骤

1. 运行 `node tools/self-check.js`
2. 在微信开发者工具导入项目
3. 确认“选饭”“做饭”两个 tab 都能抽取、切换后仍保留各自最后一次结果
4. 确认“不算！再抽一次！”会删除当前抽取记录，“考虑下...再抽一次！”会保留当前记录
5. 确认“记入记录”进入编辑页后仅预填，保存后会显示“已记入”，且不能重复记入
6. 确认管理页新增/编辑/删除可用
7. 确认记录页和抽取历史页可按北京时间展示，且历史页筛选正常
